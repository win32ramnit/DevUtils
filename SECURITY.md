# Security Policy

## Reporting a Vulnerability

If you discover a security issue in DevUtils, especially in the crypto or validation modules, please do
not open a public issue first.

Report the issue privately to the repository maintainer with:

- a description of the issue
- affected classes or methods
- reproduction steps or proof of concept
- suggested remediation if available

The maintainer should acknowledge the report, validate the issue, and coordinate a fix before public
disclosure.

## Scope

Security-sensitive areas include:

- `org.manish.utilix.crypto`
- configuration handling for encryption keys
- masking and data-redaction helpers
